# Game Design Doc

## Camera

- Top-down perspective, fixed pitch ~55°
- Camera follows the player at a fixed offset
- Optional WASD edge-pan disabled in v1 (camera is locked to player) — can add later
- Mouse wheel: zoom in/out within clamps `[15, 35]` units of distance

## Controls

| Input | Action |
|---|---|
| Left-click on ground | Move to that position |
| Left-click on enemy | Move into basic-attack range, then auto-attack |
| Left-click on item | Move to it, pick it up |
| Right-click on ground | Same as left-click on ground (move) |
| Right-click on enemy | Same as left-click on enemy (attack-move) |
| `A` then left-click | Attack-move: move to clicked location, attacking anything in range along the way |
| `S` | Stop all current actions |
| `Q` | Cast ability slot 1 |
| `W` | Cast ability slot 2 |
| `E` | Cast ability slot 3 |
| `R` | Cast ability slot 4 (passives are not bound) |
| `Tab` | Toggle inventory |
| `C` | Toggle character sheet |
| `M` | Toggle minimap |
| `B` | Toggle stash (in town only) |
| `Esc` | Open menu |
| `1`–`5` | Quick consumable hotbar (reserved for future) |
| Hold shift + ability | Cast at cursor without facing change |

### Click target rules

- Ground click: pathfinds via flat-world straight-line + obstacle avoidance
- Enemy click: pathfinds toward enemy, stops at `basicAttackRange` (default 6 units, melee 2)
- Item click: pathfinds toward item, picks up at distance `< 1.5`
- Click + already in range: triggers immediately, no pathing

### Targeted ability rules

- Single-target abilities (`spell.target == 'enemy'` or `'ally'`): require a target under cursor; if none, snap to nearest valid in 12-unit cone in front
- Self abilities (`spell.target == 'self'`): cast immediately, no target needed
- AoE abilities (`spell.target == 'all_enemies'` etc.): show ground reticle following cursor; click to cast at that location; press key again to confirm (or release-to-cast — choose one and stick: **release-to-cast** for fast play)

## Combat rules

- **Basic attack**: free, no cooldown, repeats while target is in range and alive
  - Damage formula: `attack` (the hero's `attack` stat from `hero.js`) — the existing system already scales `attack` from mainstat, so basic-attack damage scales with mainstat for free
  - Speed: attack interval = `2.0 / (actionBarSpeed / 200)` seconds (so a unit at 200 actionBarSpeed attacks every 2s, at 400 every 1s)
  - Range: `basicAttackRange` per class — melee (str/agi mainstat) defaults 2 units; ranged (int mainstat or `ranger`/`archer`/`hunter` family) defaults 14 units
  - Melee = instant damage on swing; ranged = projectile travel time
- **Abilities**: trigger spell logic via the existing `spellLogic.js` adapter
  - Cooldown converted from turns to seconds (see `04-conversion-spec.md`)
  - Speed → CDR: `effectiveCooldown = baseCooldown * 200 / actionBarSpeed`
  - All buffs/debuffs continue to function via the existing `applyBuff` / `applyDebuff` API, ticking by real seconds instead of turns
- **No mana / no resource bar** — only cooldowns gate ability use
- **No friendly fire**: AoE damage spells skip allies in radius; AoE buffs/heals catch all allies
- **Death**: HP reaches 0 → death animation → respawn at nearest town fountain after 5s

## World

- Single seamless overworld terrain, ~1024×1024 units (large enough to feel open without being PoE-vast)
- 3 town hubs spread evenly: `town_north`, `town_center`, `town_south`
- Each town: 64×64 unit safe zone, no combat, contains:
  - A fountain (respawn anchor)
  - A stash NPC (opens shared class-family stash)
  - A vendor NPC (sell items / buy potions in future)
  - A class trainer NPC (handles promotion when eligible)
  - A portal pad to the dungeon select screen
- Dungeon entrance portals scattered in overworld near their target tier:
  - Tier 0 (Easy) dungeons near town_center
  - Tier increases with distance
- Mob level: `floor(distanceToNearestTown / 50) + 1`, capped at 500 (max level)
- Mob composition near a dungeon entrance: same enemy IDs as that dungeon's waves, in groups of 3-5

## Dungeons

- Linear corridor — entrance at south end, boss/loot room at north end
- Layout: `entrance → 4 enemy rooms → mini-boss room → 2 enemy rooms → boss room → loot room`
- Each enemy room corresponds to one wave from `dungeons.json` (mob composition reused exactly)
- Mini-boss = a duplicated boss enemy from the dungeon, level slightly lower than final boss
- Final boss = the dungeon's `boss` field
- Loot room: empty room with a single item dropped on the ground, generated by:
  - Pick random `id` from dungeon's `rewards.items`
  - `new Item(id)` (existing item.js generation runs)
  - Item visual = colored glow column on ground, color matches item rarity
  - Walk over to pick up; goes into player inventory
- Dungeon clear: gives the dungeon's `rewards.gold` and `rewards.exp` on boss kill
- Re-entry: leaving the loot room teleports you back to the entrance portal in overworld
- Each dungeon entrance is on a per-character cooldown of 0 (re-runnable immediately) — gating is via tier unlock

## Progression

- Same as original:
  - Tier 0 → Tier 1 at level 50, costs 1k gold
  - Tier 1 → Tier 2 at level 100, costs 10k
  - Tier 2 → Tier 3 at level 200, costs 100k
  - Tier 3 → Tier 4 at level 300, costs 1M
  - Tier 4 awakening at level 400, costs 10M
- Promotion happens at the class trainer NPC in town
- Items/stash carry through promotions within the same family (existing behavior)

## Character roster

- 10 character slots per save
- Character select screen on game start
- Each character is fully independent: own gear, own level, own progression
- Stashes are shared per **class family**, across all characters of that family (existing rule)
- "Active character" is the one currently in the world
- Swapping characters mid-game:
  - In a town: free swap, instant
  - In overworld: kills current character (drops nothing — anti-cheese), respawns new character at nearest town
  - In a dungeon: kills current character, ejects from dungeon, new character spawns at town

## Multiplayer

- 6-player lobby max
- Host creates lobby; friends join via room code
- Joining: pick which character slot to load; that character spawns at host's current location
- All players share host's overworld instance and dungeon instance
- Loot in dungeons:
  - Each player gets their **own** loot room instance — when host enters loot room, server generates a per-player item drop using each player's character class as a small influence on weights (or just random for v1)
  - Loot is **not** ground-shared (avoids drama)
- Friendly fire: OFF
- Voice / text chat: text chat only in v1, top-right input bar

## Death penalty

- Respawn at nearest town fountain after 5s
- No XP loss, no gold loss, no item drop
- Hardcore mode TBD (post-v1)

## Visual / aesthetic

- Stand-in 3D models: simple capsules + cylinder for arms + sphere for head, color-tinted by class family
  - Acolyte: white
  - Archer: green
  - Druid: brown
  - Initiate: blue
  - Swordsman: red
  - Templar: yellow
  - Thief: purple
  - Witch Hunter: orange
  - Villager: gray
- Enemies: same primitive style, tinted by `enemy.boss ? red : darkgray`, scaled by tier
- Terrain: single procedural mesh with simple grass-green Lambert material; town squares are stone-gray patches
- Dungeon interiors: dark gray stone corridor with wall meshes
- AoE telegraphs: red ring on ground for damage, green for buffs/heals, with fill-in animation matching cast time
- Damage numbers: floating text from impact point, color-coded (white = physical, blue = magic, green = heal, yellow = crit)
- Status effects: small icons floating above hero head with duration ticks
